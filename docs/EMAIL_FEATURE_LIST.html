OpenPiBot Multi-Agent Dashboard — Feature List

Ken,

Attached is the proof-of-concept package for the Raspberry Pi OpenPiBot dashboard. It includes the app source, sanitized config example, systemd service, proposal writeup, install/operations notes, API reference, security notes, and feature list.

What the proof of concept demonstrates

  • Raspberry Pi as an AI control node: OpenPiBot runs on a Pi and exposes an OpenAI-compatible control interface.
  • Multi-agent dashboard: One browser page monitors and controls multiple OpenPiBot-compatible endpoints.
  • Device/software control path: The same pattern can control local programs, Pi Pico projects, sensors, robotic arms, Blender/design workflows, and plant-floor devices.
  • Hermes-like control layer: The architecture points toward a lightweight, custom agent-control CLI/dashboard for moving quickly from device to device.

Dashboard features

  • Online/offline health cards for each configured agent.
  • Continuous polling of /health and /v1/models.
  • Chat / Control Console for one selected agent or all online agents.
  • Persistent per-agent session context.
  • New Session, Clear Window, and Clear Context controls.
  • Stop button for active requests.
  • Command / Activity Monitor with persistent redacted logs.
  • Manage Agents UI for adding, testing, saving, updating, and removing bots without SSH/JSON editing.
  • Server-side API key storage; saved keys are not returned to the browser.
  • Systemd boot startup so the dashboard comes back after Pi reboot.

Potential plant-management uses

  • Plant device status dashboard.
  • Sensor monitoring and troubleshooting.
  • Robotic arm or machine-control assistant with safety rules.
  • Pi Pico/microcontroller programming station.
  • PC application automation for drawing, CAD, Blender, or digital-printing workflows.
  • Maintenance assistant for diagnostics, service resets, logs, and checklists.

Recommended next steps

  1. Add dashboard login/authentication.
  2. Create an installer script for fresh Raspberry Pis.
  3. Add device/agent groups by plant, line, room, or cell.
  4. Create agent templates for Pi, workstation, sensor, robot, and fabrication use cases.
  5. Add approval/safety rules for dangerous commands.
  6. Pilot with 2–5 real devices in a plant/lab environment.

Security note: The package is sanitized. It does not include live API keys, live endpoint config, chat sessions, or logs.
